## What's in here

Two ways to materialize a SQL query result as a DataFrame in Python — pandas and polars — using parameterized queries to defend against SQL injection.

- `read_sql.py` — the demo script
- `prices.db` — sample SQLite database with a `prices` table (14 tickers × daily OHLCV)

## Run it

```bash
pip install -r requirements.txt
python read_sql.py
```

Output:

```
pandas read_sql:      (367, 8)
polars read_database: (367, 8)
```

## What it shows

1. `pd.read_sql(SQL, conn, params=PARAMS)` — pandas reads the cursor.
2. `pl.read_database(SQL, conn, execute_options={"parameters": PARAMS})` — polars wraps the same DB-API connection.

Both arrive at the same shape. The `?` placeholders + `params` tuple is the textbook defense against SQL injection — let the driver escape values, never f-string user input.
